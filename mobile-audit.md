# 移动端 UI 审查问题清单

## 问题 1: Home 页面与 Chat 页面割裂
- 移动端 Home 是纯展示页，点"开始检测"跳到 /chat
- /chat 才出现 TabBar，体验断裂
- 解决：移动端首页直接合并检测功能，展示上传区+快捷问题+TabBar

## 问题 2: "和芯颜AI聊天"按钮太敷衍
- 只是一个普通按钮，用 window.location.href 跳转
- 跳转不流畅，没有过渡动画
- 解决：重新设计为醒目的浮动按钮/卡片，使用 setLocation 路由跳转

## 问题 3: Result 页面移动端展示差
- 左右分栏在移动端变成上下堆叠，评分区占太多空间
- 底部操作栏按钮太多太挤
- Tab 内容区域太小
- 解决：移动端重新设计为紧凑的评分头部+全宽 Tab 内容

## 设计方案
1. Home 移动端：保留简洁 hero 区域但加入上传入口和快捷问题，底部有 TabBar
2. Result 移动端：评分做成紧凑卡片头部，6 维度横向滚动，Tab 全宽展示
3. Chat 按钮：在 Result 底部做成醒目的渐变浮动按钮
